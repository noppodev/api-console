API 概要

NoppoAPI は、NoppoStudio の各サービスを単一のエンドポイントで統合管理するためのデベロッパー向けゲートウェイです。

すべての API リクエストのベース URL は以下の通りです：

https://noppo-api.noppo5319.workers.dev
注意: すべてのリクエストは HTTPS 経由である必要があります。HTTP リクエストは自動的に 301 リダイレクトされます。

認証方法

NoppoAPI のすべてのエンドポイントは API キーによる認証を必要とします。API キーはコンソールから発行してください。

リクエストヘッダー

ヘッダー名 値 説明
noppo-api-key np_... 発行された API キーを指定してください。必須
Content-Type application/json POST リクエストの際に指定してください。

NoppoIntelligence

対話型 AI をアプリケーションに組み込むためのサービスです。

POST /v1/ai/chat

リクエストボディ

パラメータ 型 説明
message string AI に送信するテキスト内容。必須
agent_mode boolean Web 検索を使用するかどうか（デフォルト: false）。

実装例

JavaScript
cURL
Response
const res = await fetch("https://noppo-api.noppo5319.workers.dev/v1/ai/chat", {
  method: "POST",
  headers: {
    "noppo-api-key": "YOUR_API_KEY",
    "Content-Type": "application/json"
  },
  body: JSON.stringify({ message: "こんにちは" })
});
const data = await res.json();
curl -X POST https://noppo-api.noppo5319.workers.dev/v1/ai/chat \
  -H "noppo-api-key: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"message": "こんにちは"}'
{
  "status": "success",
  "content": "こんにちは！NoppoIntelligenceです。何かお手伝いしましょうか？",
  "model": "noppo-v3-pro"
}

NoppoDrive

クラウドストレージのファイル情報を操作・取得します。

GET /v1/drive/list

認証されたユーザーのファイル・ディレクトリ構造をリスト形式で取得します。

JSON Response
[
  {
    "id": "f_01",
    "name": "ProjectDocs",
    "type": "folder",
    "updated": "2026-03-01T12:00:00Z"
  }
]

エラーコード

API リクエストの結果、以下のステータスコードが返される場合があります。

コード ステータス 理由
400 Bad Request パラメータが不足しているか、形式が正しくありません。
401 Unauthorized API キーが指定されていないか、誤っています。
403 Forbidden 指定された API キーには権限がありません。
429 Too Many Requests レート制限を超過しました。しばらく待ってから再試行してください。
500 Internal Server Error サーバー内部で予期しないエラーが発生しました。
