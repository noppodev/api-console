API Reference v1

本ドキュメントは、NoppoStudio Infrastructureにおける統合APIエンドポイントの技術仕様を記述したものである。すべてのリクエストは、NoppoAPIConsoleにて発行された認証資格情報を要求する。

認証 (Authentication)

全てのAPIリクエストには、HTTPヘッダーに有効な noppo-api-key を含める必要がある。このキーは np_ プレフィックスから始まる構造化文字列であり、NoppoAPIConsoleを通じてのみ発行および管理が可能である。

Header Type Description
noppo-api-key String Required 発行されたAPIキー。

drive.files.list

認証されたアカウントに関連付けられたファイルリソースのメタデータ一覧を取得する。レスポンスはページネーションに対応したJSONオブジェクトとして返却される。

GET https://noppo-api.noppo5319.workers.dev/v1/drive/files

Query Parameters

Parameter Type Description
folder_id string 取得対象の親ディレクトリ識別子。指定がない場合はルートディレクトリが対象となる。
pageSize integer 一度のリクエストで返却される最大アイテム数（デフォルト: 20）。

Response Body

{
  "kind": "noppo#fileList",
  "files": [
    {
      "id": "string",
      "name": "string",
      "mimeType": "string",
      "size": "long"
    }
  ]
}

intelligence.chat.completions

Noppo Intelligence (Nexus) LLM への推論リクエストを実行する。入力メッセージの履歴を保持したコンテキスト処理が可能である。

POST https://noppo-api.noppo5319.workers.dev/v1/ai/chat

Request Body

Field Type Description
messages array Required roleおよびcontentを含むメッセージオブジェクトのリスト。
stream boolean ストリーミングレスポンスの有効化。デフォルトはfalse。

Code Sample (cURL)

curl -X POST "https://noppo-api.noppo5319.workers.dev/v1/ai/chat" \
     -H "noppo-api-key: np_xxxxxxxxxxxx" \
     -H "Content-Type: application/json" \
     -d '{"messages": [{"role": "user", "content": "Query"}]}'
